Step 4: Currying

With a concept called currying (also created by Haskell Curry), we can transform functions with any number of arguments into a chain of functions each taking only one.

So if we have a function like this:

function f (x, y) {
   ...
};

Which is called like this:

f(x, y);

We can transform it, using currying, into a function like this:

function F (x) { return function (y) {
   ...
}};

And call it like this:

F(x)(y);

The transformed function chains it's input into multiple returns of subsequent functions. This is a very poweful tool when used with higher-order functions, because it allows the programmer to create dynamic toolchains made up of smaller functions working together as opposed to one static function which takes a specific input.
